# Contributing to Infra Operations

Thank you for considering contributing to the Infra Operations project! We appreciate your efforts to improve the codebase. Please follow the guidelines below.

## Code of Conduct
We expect all contributors to adhere to our code of conduct. Please be respectful and inclusive in discussions. Any form of harassment or discrimination will not be tolerated.

## Reporting Issues
If you find a bug or have a suggestion, please report it by creating an issue in the repository. Include the following information:
- A clear and descriptive title.
- A detailed description of the issue or suggestion.
- Steps to reproduce the problem (if applicable).

## Feature Requests
We welcome feature requests! To suggest a new feature, please open an issue with the label "feature request" and describe your idea clearly.

## Code Contributions
If you would like to contribute code:
1. Fork the repository.
2. Create a new feature branch from `main`.
3. Make your changes in that branch.
4. Submit a pull request describing your changes and why they should be merged.

## Git Workflow
Here is a simple workflow to follow:
- Clone your fork of the repo:
  ```bash
  git clone https://github.com/tharindut-wso2/infra-operations.git
  cd infra-operations
  ```
- Create a new branch:
  ```bash
  git checkout -b feature/my-feature
  ```
- Commit your changes:
  ```bash
  git commit -m "Add my feature"
  ```
- Push your branch:
  ```bash
  git push origin feature/my-feature
  ```
- Open a pull request.

## Code Style Guidelines
We follow specific coding standards for different languages:
- **Python:**
  - Follow PEP 8 style guide.
- **PHP:**
  - Follow PSR-12 standards.
- **Shell:**
  - Adhere to best practices in shell scripting. Use "shellcheck" to validate.

## Testing Requirements
Before submitting a pull request, ensure that your changes are thoroughly tested. Write unit tests for new features and ensure existing tests pass.

## Security Practices
Before contributing:
- Ensure that your code does not introduce security vulnerabilities.
- Validate and sanitize user inputs.
- Use secure coding practices throughout your contributions.

## Development Setup Instructions
1. Clone the repository:
```bash
git clone https://github.com/tharindut-wso2/infra-operations.git
cd infra-operations
```
2. Install dependencies:
   - Follow the installation instructions provided in the repository.
3. Start the development environment:
   - Set up according to project guidelines.

Thank you for contributing!
